# KinEcho 数据库表结构说明

本文档说明当前后端初始化时创建的 MySQL 数据表。建表入口位于 `server/app.py` 的 `init_db()`，数据库连接配置位于 `server/.env`。

## 1. 数据库配置说明

当前项目默认使用 MySQL：

```env
DB_DRIVER=mysql
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_DATABASE=kinecho
MYSQL_AUTO_CREATE_DATABASE=true
SEED_DEMO_DATA=true
```

说明：

- `MYSQL_AUTO_CREATE_DATABASE=true` 时，后端启动会自动创建数据库。
- `SEED_DEMO_DATA=true` 时，后端会写入一组演示数据，避免老人端首屏为空。
- 文档不记录数据库密码，密码只应保存在本机 `.env` 文件中。
- 代码中保留 SQLite 兜底模式，仅用于本地烟测或临时调试。

## 2. 表关系概览

核心业务围绕 `family_id` 组织，一个家庭组下包含老人、家属、护理计划、留言、媒体、情绪记录和咨询记录。

主要关系：

- `users` 保存老人和家属账号，通过 `family_id` 归属同一个家庭。
- `schedules` 保存家属创建的护理计划，老人端读取今日提醒。
- `family_messages` 保存家属给老人的留言，老人端可播放和点赞。
- `family_alerts` 保存老人端求助、联系家人、系统提醒等通知。
- `media` 保存家属上传的照片和视频。
- `media_policies` 控制媒体推荐策略。
- `media_play_history` 保存老人端播放历史。
- `media_feedback` 保存老人对媒体的喜欢或不喜欢反馈。
- `mood_records` 保存老人每日情绪和状态记录。
- `avatar_interactions` 保存老人和数字人陪伴助手的对话历史。
- `counselors` 保存心理咨询师资料。
- `consultations` 保存心理咨询预约和历史记录。

## 3. 表清单

| 表名 | 用途 |
| --- | --- |
| `users` | 用户表，保存老人和家属基础信息 |
| `schedules` | 护理计划或日程表 |
| `reminders` | 提醒记录表，预留给细粒度提醒状态 |
| `family_messages` | 家属留言表 |
| `family_alerts` | 家属端通知和告警表 |
| `mood_records` | 老人情绪记录表 |
| `media` | 媒体文件表 |
| `media_tags` | 媒体标签表 |
| `media_policies` | 媒体推荐策略表 |
| `media_play_history` | 媒体播放历史表 |
| `media_feedback` | 媒体反馈表 |
| `avatar_interactions` | 数字人对话历史表 |
| `counselors` | 心理咨询师表 |
| `consultations` | 心理咨询预约和记录表 |

## 4. 详细字段说明

### 4.1 `users`

用户表，保存老人和家属的基础信息。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `user_type` | 用户类型，`family` 表示家属，`elderly` 表示老人 |
| `name` | 用户姓名 |
| `phone` | 手机号 |
| `family_id` | 家庭组 ID，用于关联老人和家属 |
| `created_at` | 创建时间 |

### 4.2 `schedules`

护理计划或日程表，家属端创建，老人端展示今日提醒。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `family_id` | 家庭组 ID |
| `title` | 日程标题 |
| `description` | 详细说明 |
| `schedule_type` | 类型，如 `medication`、`exercise`、`meal`、`checkup`、`other` |
| `schedule_time` | 计划时间 |
| `repeat_type` | 重复类型，如 `once`、`daily`、`weekly`、`monthly` |
| `repeat_days` | 每周重复日期，JSON 字符串，如 `[1,3,5]` |
| `status` | 状态，如 `pending`、`completed`、`skipped`、`missed` |
| `completed_at` | 完成时间 |
| `auto_remind` | 是否启用数字人自动提醒，1 表示启用 |
| `is_active` | 是否有效，软删除使用 |
| `created_by` | 创建者用户 ID |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.3 `reminders`

提醒记录表，预留给每次具体提醒的状态追踪。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `schedule_id` | 对应的护理计划 ID |
| `elderly_id` | 老人用户 ID |
| `remind_time` | 提醒时间 |
| `status` | 提醒状态，如 `pending`、`completed`、`missed`、`dismissed` |
| `completed_at` | 完成时间 |
| `created_at` | 创建时间 |

### 4.4 `family_messages`

家属留言表，家属端创建，老人端读取、播放、点赞。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `family_id` | 家庭组 ID |
| `content` | 留言内容 |
| `sender_name` | 发送者姓名 |
| `sender_relation` | 发送者关系，如女儿、儿子、孙女 |
| `scheduled_time` | 预约播放时间 |
| `played` | 是否已播放，0 未播放，1 已播放 |
| `played_at` | 实际播放时间 |
| `liked` | 老人是否点赞 |
| `is_active` | 是否有效，软删除使用 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.5 `family_alerts`

通知和告警表，用于老人端求助、联系家人、媒体展示事件和系统提醒。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `family_id` | 家庭组 ID |
| `elderly_id` | 老人用户 ID，可为空 |
| `alert_type` | 通知类型，如 `sos_emergency`、`contact_family`、`emotion`、`media_display` |
| `level` | 级别，如 `low`、`medium`、`high` |
| `title` | 通知标题 |
| `message` | 通知内容 |
| `metadata` | JSON 字符串，保存扩展信息 |
| `source` | 来源，如 `elderly`、`system`、`family` |
| `handled` | 是否已处理 |
| `handled_at` | 处理时间 |
| `handled_by` | 处理人用户 ID |
| `reply_message` | 家属回复内容 |
| `read` | 是否已读。MySQL 中代码会自动转义该字段 |
| `read_at` | 阅读时间 |
| `is_active` | 是否有效，软删除使用 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.6 `mood_records`

老人情绪记录表，老人端“今日记录”页面写入，家属端可查看趋势。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `family_id` | 家庭组 ID |
| `elderly_id` | 老人用户 ID |
| `mood_type` | 情绪类型，如 `happy`、`calm`、`sad`、`anxious`、`angry`、`tired` |
| `mood_score` | 情绪分数，1 到 10，数字越大越积极 |
| `note` | 备注，比如睡眠、食欲、活动、服药信息 |
| `source` | 来源，如 `manual`、`ai_detect`、`voice` |
| `trigger_event` | 触发事件或今日活动 |
| `location` | 记录地点 |
| `weather` | 天气 |
| `recorded_at` | 记录时间 |
| `created_at` | 创建时间 |

### 4.7 `media`

媒体文件表，保存家属端上传的照片和视频信息。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `family_id` | 家庭组 ID |
| `media_type` | 媒体类型，`photo` 或 `video` |
| `title` | 媒体标题 |
| `description` | 媒体描述 |
| `file_path` | 文件存储路径 |
| `file_size` | 文件大小，单位字节 |
| `duration` | 视频时长，照片为空 |
| `thumbnail_path` | 缩略图路径 |
| `uploaded_by` | 上传者用户 ID |
| `is_active` | 是否有效，软删除使用 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.8 `media_tags`

媒体标签表，一个媒体可以有多个标签。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `media_id` | 媒体 ID |
| `tag` | 标签内容，如生日、旅行、孙女 |
| `created_at` | 创建时间 |

### 4.9 `media_policies`

媒体推荐策略表，用于控制老人端回忆推荐。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `media_id` | 媒体 ID |
| `time_windows` | 播放时段，JSON 字符串，如 `["07:00-09:00"]` |
| `moods` | 适合情绪，JSON 字符串 |
| `occasions` | 特殊场景，JSON 字符串 |
| `cooldown` | 冷却时间，单位分钟 |
| `priority` | 推荐优先级，数字越大越优先 |
| `last_played_at` | 上次播放时间 |
| `play_count` | 播放次数 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.10 `media_play_history`

媒体播放历史表，老人端播放回忆时写入。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `media_id` | 媒体 ID |
| `elderly_id` | 老人用户 ID |
| `played_at` | 播放时间 |
| `duration_watched` | 观看时长，单位秒 |
| `completed` | 是否看完 |
| `triggered_by` | 触发方式，如 `auto`、`manual`、`mood` |
| `mood_before` | 播放前情绪 |
| `mood_after` | 播放后情绪 |

### 4.11 `media_feedback`

媒体反馈表，保存老人对媒体的喜欢或不喜欢。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `media_id` | 媒体 ID |
| `elderly_id` | 老人用户 ID |
| `feedback_type` | 反馈类型，`like` 或 `dislike` |
| `created_at` | 创建时间 |

约束：

- `UNIQUE(media_id, elderly_id)`，同一老人对同一媒体只保留一条反馈。

### 4.12 `avatar_interactions`

数字人陪伴对话历史表，记录老人和数字人助手的聊天内容。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `username` | 对话用户名，默认 `User` |
| `type` | 消息类型，如 `member` 表示老人，`fay` 表示数字人 |
| `way` | 交互方式，默认 `speak` |
| `content` | 消息内容 |
| `createtime` | 毫秒时间戳 |
| `timetext` | 北京时间文本 |
| `created_at` | 创建时间 |

### 4.13 `counselors`

心理咨询师表，老人端和家属端预约咨询使用。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `name` | 咨询师姓名 |
| `title` | 咨询师职称 |
| `experience` | 从业经验 |
| `specialty` | 擅长领域 |
| `rating` | 评分 |
| `avatar` | 头像文字或标识 |
| `available` | 是否可预约 |
| `is_active` | 是否有效 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.14 `consultations`

心理咨询预约和记录表。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键，自增 |
| `family_id` | 家庭组 ID |
| `elderly_id` | 老人用户 ID |
| `counselor_id` | 咨询师 ID |
| `consultation_type` | 咨询方式，如 `phone`、`video`、`text` |
| `scheduled_time` | 预约时间 |
| `duration` | 预计咨询时长，单位分钟 |
| `status` | 状态，如 `scheduled`、`completed`、`cancelled` |
| `note` | 备注 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

## 5. 索引说明

初始化时创建以下索引，用于提升常用查询性能：

| 索引名 | 表 | 字段 | 用途 |
| --- | --- | --- | --- |
| `idx_mood_records_family_id` | `mood_records` | `family_id` | 按家庭查询情绪记录 |
| `idx_mood_records_elderly_id` | `mood_records` | `elderly_id` | 按老人查询情绪记录 |
| `idx_mood_records_recorded_at` | `mood_records` | `recorded_at DESC` | 按时间倒序查询情绪记录 |
| `idx_family_alerts_family_id` | `family_alerts` | `family_id` | 按家庭查询通知 |
| `idx_family_alerts_created_at` | `family_alerts` | `created_at DESC` | 按创建时间倒序查询通知 |
| `idx_family_alerts_handled` | `family_alerts` | `handled, created_at DESC` | 查询未处理或已处理通知 |

## 7. 额外说明

项目中还有一个 `T_Msg` 表，它属于 Fay 数字人外部工具链的本地记忆库，不属于当前 KinEcho MySQL 主业务库。当前 KinEcho 主业务库以本文档列出的表为准。
