# 数据库表结构说明

当前后端由 `server-java` 提供，建表入口在 `server-java/src/main/java/com/kinecho/server/mapper/KinEchoMapper.java`。后端采用 MyBatis-Plus 作为 mapper 层，包路径为 `server-java/src/main/java/com/kinecho/server/mapper`。

当前持久化数据统一使用 MySQL。连接信息不再通过环境变量配置，而是直接写在 Spring Boot 配置文件中：

- `server-java/src/main/resources/application-local.yml`：本地开发数据库
- `server-java/src/main/resources/application-prod.yml`：生产数据库

示例：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://127.0.0.1:3306/kinecho?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: your-mysql-password
```

如果后续出现缓存需求，例如临时会话、短期统计或限流计数，可以使用 Redis；不要把核心业务数据放进 Redis。

## 核心表

| 表名 | 用途 |
| --- | --- |
| `users` | 老人、家属、服务人员等用户档案 |
| `schedules` | 日程、用药、运动、复诊等提醒 |
| `family_alerts` | 老人端求助和系统预警 |
| `mood_records` | 情绪记录 |
| `elderly_messages` | 家属给老人端的定时留言 |
| `media` | 家属上传的照片、视频等回忆内容 |
| `media_tags` | 媒体标签 |
| `media_policies` | 媒体推荐策略 |
| `media_play_history` | 老人端播放记录 |
| `media_feedback` | 老人端喜欢/不喜欢反馈 |
| `counselors` | 咨询师或服务人员资源 |
| `consultations` | 咨询或服务预约 |
| `service_tasks` | 服务人员任务 |
| `ai_interactions` | 老人与 AI 陪伴助手的对话历史 |

## AI 对话历史

`ai_interactions` 记录小程序老人端的语音或文字对话。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `username` | 对话用户标识 |
| `type` | `member` 表示老人输入，`ai` 表示 AI 回复 |
| `way` | 输入方式，例如 `text` |
| `content` | 消息内容 |
| `createtime` | 毫秒时间戳 |
| `timetext` | 可读时间文本 |

语音文件和合成音频默认保存在 `server/uploads/`，该目录属于本地运行数据，不进入版本库。
