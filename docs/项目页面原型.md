请为我设计一整套完整的、可正式上线的“老年心理健康陪护平台”UI 界面原型，要求达到企业级产品交付标准，而不是概念稿、学生作业稿或少量示意页。请直接生成完整的多角色产品原型，包括：

1. 老人陪伴端（移动端小程序）
2. 家属照护端（移动端小程序）
3. 服务协同端（移动端/轻管理端，适用于医生、心理咨询师、社工、护理人员）

整套产品需要围绕“老人心理健康、家庭远程照护、专业服务协同、企业级运营管理”构建完整闭环，能够体现真实产品逻辑、正式上线视觉质量、完整信息架构、完整功能模块和多端统一设计系统。

--------------------------------
一、项目定位
--------------------------------

这是一个面向老年人及其家庭、并可接入社区/机构/医疗服务的心理健康陪护平台，核心业务目标包括：

1. 帮助老人记录情绪与健康状态，获得 AI 数字人陪伴、家庭回忆内容、提醒任务和快捷求助能力；
2. 帮助家属远程查看老人状态，发送留言，安排护理计划，管理记忆内容，接收并处理风险预警；
3. 帮助专业服务人员（心理咨询师、社工、医生、护理人员）查看重点对象、记录干预、随访老人、承接转介工单；
4. 帮助平台运营管理人员管理用户、内容、规则、风险事件、服务数据和系统配置。

请将这套产品理解为一个完整的企业级多角色协同系统，而不是单纯的陪伴类小程序。

--------------------------------
二、设计总原则
--------------------------------

【整体风格关键词】
温暖、安心、专业、可信、成熟、适老化、医疗健康感、家庭陪伴感、企业级管理感

【视觉要求】
1. 不要玩具化，不要过度卡通，不要娱乐化社交 App 风格；
2. 老人端要极致适老化，大字号、大按钮、高对比、低认知负担；
3. 家属端要体现家庭照护和远程管理感；
4. 服务协同端要体现专业服务与任务处理感；
5. 后台管理端要体现标准企业级管理后台风格，清晰、规范、专业。

【色彩建议】
- 主色：柔和蓝绿色、医疗感绿色、安心暖蓝
- 辅助色：米白、浅灰、低饱和暖色

- 背景：浅色系，保持清爽和长期使用舒适度

【组件风格】
- 卡片式布局
- 清晰的状态标签
- 大按钮、标准按钮、危险按钮、次要按钮
- 图表卡片简洁专业
- 表单规范完整
- 列表结构清晰
- 空状态、异常状态、确认弹窗完整
- 多端组件统一但可因角色而做风格差异化

【原型输出要求】
1. 直接输出完整页面原型，不要只给少量示意页；
2. 每个端都要有完整页面结构、导航、详情页、编辑页、空状态页、弹窗页；
3. 页面应接近真实上线版本的高保真原型；
4. 所有页面保持统一设计系统；
5. 所有模块都有合理的业务闭环。
