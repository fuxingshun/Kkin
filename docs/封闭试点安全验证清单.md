# 封闭试点安全验证清单

适用范围：封闭试点上线前，在预生产或隔离开发库执行。所有测试账号、家庭档案和上传文件应使用测试数据，不使用真实老人隐私数据。

## 环境前置

- Java 后端使用 `prod` 或等价安全配置启动。
- `KINECHO_API_TOKEN_ENABLED=true`，并只向小程序、管理端和受控脚本分发 Token。
- `KINECHO_FAMILY_SCOPE_SESSION_REQUIRED=true`。
- `KINECHO_PHONE_SUFFIX_LOGIN_ENABLED=false`。
- 至少准备两个家庭档案：`family_001` 和 `family_002`，每个家庭至少有 1 位老人和 1 位家属。
- 准备老人端、家属端、服务端、管理端各 1 个测试账号。

## 会话与家庭隔离

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| SEC-01 | 未登录访问家庭数据 | 不带 `X-KinEcho-Session` 请求 `/api/family/messages?family_id=family_001` | 返回 401，不返回家庭数据 |
| SEC-02 | 跨家庭查询 | 使用 `family_001` 家属 session 请求 `/api/family/messages?family_id=family_002` | 返回 403，不返回 `family_002` 数据 |
| SEC-03 | 跨家庭写入 | 使用 `family_001` 家属 session 向 `/api/family/messages` 写入 `family_id=family_002` | 返回 403，`family_002` 无新增记录 |
| SEC-04 | 管理端授权访问 | 使用 admin session 切换查看 `family_001` 与 `family_002` | 管理端可按授权选择家庭，操作写入审计 |
| SEC-05 | 无效 session | 修改 `X-KinEcho-Session` 任意字符后请求家庭接口 | 返回 401 |

## Token 与默认口令

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| SEC-06 | 缺失 API Token | 在 `KINECHO_API_TOKEN_ENABLED=true` 时不带 `X-KinEcho-Token` 请求非公开接口 | 返回 401 |
| SEC-07 | 错误 API Token | 使用错误 Token 请求 `/api/me` | 返回 401 |
| SEC-08 | 手机号后 6 位禁用 | 生产配置下使用手机号后 6 位登录老人端或家属端 | 登录失败，提示账号或密码错误 |
| SEC-09 | 管理端默认口令禁用 | 生产配置不使用默认 `admin/123456` | 默认口令无法登录 |

## 上传与静态资源

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| SEC-10 | 禁止公开上传目录 | 直接访问 `/uploads/<历史图片文件名>` | 不返回家庭上传图片；应为 404 或受控拒绝 |
| SEC-11 | 受控媒体下载 | 使用正确家庭 session 访问 `/api/family/media/{mediaId}/file?family_id=family_001` | 可下载同家庭媒体 |
| SEC-12 | 跨家庭媒体下载 | 使用 `family_002` session 下载 `family_001` 媒体 | 返回 403 或 404 |
| SEC-13 | 上传类型限制 | 尝试上传脚本、HTML、超大文件或伪装扩展名文件 | 拒绝保存，返回明确错误 |
| SEC-14 | AI 音频签名过期 | 等待 `/api/ai/audio/{filename}?token=...` 超过 TTL 后访问 | 返回 401 或 403 |

## 隐私、审计与数据操作

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| SEC-15 | 同意记录 | 登录后记录用户协议和隐私政策同意 | `consent_records` 有版本、类型、用户和家庭记录 |
| SEC-16 | 家庭数据导出 | 家属端提交家庭数据导出 | 导出内容只包含当前家庭数据，并写入照护审计 |
| SEC-17 | 删除/更正请求 | 家属端提交删除或更正请求，管理端审核 | `privacy_requests` 状态变化，审核动作写入审计 |
| SEC-18 | 高风险事件审计 | 触发 AI 危机关键词 | 生成高优先级家庭提醒和 `ai_crisis_detected` 审计记录 |
