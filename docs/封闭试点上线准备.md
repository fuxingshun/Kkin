# 封闭试点上线准备

本文档用于冻结 KinEcho 首批封闭试点的上线边界、参与名单和产品安全表述。首版仅面向受邀家庭，不做公众开放。

## 试点边界

- 试点规模：10-30 户真实家庭。
- 服务配置：1-2 名服务人员，固定咨询师资源。
- 核心路径：老人触发或系统识别异常 -> 家属通知 -> 服务人员处理 -> 家属看到处理结果。
- 每日检查：高风险工单、AI 失败率、老人使用率、家属查看率、咨询完成率，执行表见 [封闭试点每日运营检查表](封闭试点每日运营检查表.md)。

## 非医学诊断表述

对用户、家属、服务人员和管理端统一使用以下定位：

- KinEcho 提供陪伴、提醒、家庭沟通和照护协同支持。
- 心理关怀、情绪记录、AI 陪伴和现场采集结果仅用于健康关怀与风险筛查参考。
- KinEcho 不提供医学诊断、治疗建议或急救替代方案。
- 出现自伤、轻生、失联、跌倒或其他紧急风险时，应立即联系家属、服务人员或当地急救渠道。

禁止在页面、运营物料、客服话术中使用以下承诺：

- 诊断抑郁、焦虑或其他疾病。
- 自动判断老人真实心理状态。
- 替代医生、心理咨询师、急救或家属照护。
- 保证识别所有高风险事件。

## 上线前检查

- `npm run build:admin` 通过。
- `npm --prefix miniapp run build:weapp` 通过。
- 小程序构建输出 `check-weapp-bundle-budget` 通过，主包和各分包不超过当前预算。
- `mvn test -f server-java/pom.xml` 通过。
- `npm --prefix miniapp run smoke:checklist` 通过。
- 按 [封闭试点安全验证清单](封闭试点安全验证清单.md) 完成跨家庭、未登录、Token、上传和隐私审计验证。
- 生产环境关闭手机号后 6 位演示登录：`KINECHO_PHONE_SUFFIX_LOGIN_ENABLED=false`。
- 生产环境开启家庭会话鉴权：`KINECHO_FAMILY_SCOPE_SESSION_REQUIRED=true`。
- 上传媒体使用受控下载地址，不暴露公开 `/uploads/**`。
- 高风险 AI 陪伴命中后生成家庭提醒和照护审计。
